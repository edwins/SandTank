Contributing

We welcome your contributions to the development of SandTank. This document will help you with the process.

Before You Start

For contributing to the client, please follow the coding style:

  • Follow Airbnb JavaScript Style Guide.
  • Use soft-tabs with a two space indent.
  • Don’t put commas first.

Workflow

  1. Fork hydroframe/SandTank.

  2. Clone the repository to your computer and install dependencies.

  3. Create a feature branch.

  4. Start hacking.

  5. Use Commitizen for commit message

  6. Push the branch:

  7. Create a pull request and describe the change.

Notice

  • Don’t modify the version number in package.json. It is modified automatically.

  • Your pull request will only get merged when tests have passed. Don’t forget to run tests before submission.

Updating Documentation

The SandTank documentation is part of the code repository.

Reporting Issues

If you encounter any problems with SandTank, please check on GitHub first then ask your question in any communication channel.
